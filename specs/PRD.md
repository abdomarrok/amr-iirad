# PRD — نظام أوامر الإيراد (Amr-Iirad Mini-App)
> **المرجع القانوني:** التعليمة رقم 08 المؤرخة في 09 أبريل 2023  
> **الجهة المستفيدة:** المدرسة العليا للقضاء (ESM) — مصلحة الميزانية والمحاسبة  
> **نوع التطبيق:** تطبيق مكتبي (JavaFX 21 / MariaDB)  
> **الإصدار:** 1.0.0 — مرحلة التطوير الأولى  

---

## 1. ملخص المنتج (Executive Summary)

**الإشكالية:** يتم حالياً إنجاز أوامر الإيراد يدوياً، مما يؤدي إلى أخطاء في الحساب، صعوبة في الاسترجاع، وعدم التوحيد في الملحقات المطلوبة.

**الحل:** تطبيق مكتبي خفيف، يعمل بلا إنترنت، يرقمن دورة أمر الإيراد الكاملة من الإنشاء حتى الإرسال إلى أمين الخزينة.

**المستخدم المستهدف:** مسؤول الميزانية في ESM (مستخدم واحد أو مجموعة صغيرة في نفس الشبكة).

---

## 2. المتطلبات الوظيفية (Functional Requirements)

### 2.1 إدارة السنة المالية
- [ ] إنشاء سنة مالية جديدة (مثال: 2024)
- [ ] تفعيل/تعطيل سنة مالية
- [ ] منع إنشاء أوامر في سنة غير مفعّلة

### 2.2 إدارة المدينين (Debtors)
- [ ] إضافة / تعديل / أرشفة مدين
- [ ] بحث واسترجاع المدينين
- [ ] أنواع: فرد، شركة، هيئة عمومية

### 2.3 أمر الإيراد (Annexe 1 & 2) — المحور الرئيسي
- [ ] إنشاء أمر إيراد جديد بترقيم تلقائي (YYYY-NNN)
- [ ] ربط الأمر بالسنة المالية، المدين، ومحور الميزانية
- [ ] تعيين الموضوع بالعربية
- [ ] تحويل المبلغ إلى كلمات عربية تلقائياً (TafqeetJ)
- [ ] دورة حياة الحالات: DRAFT → ISSUED → DISPATCHED
- [ ] طباعة النموذجين (Annexe 1 و 2) بصيغة PDF عبر JasperReports

### 2.4 أمر الإلغاء / التخفيض (Annexe 3 & 4)
- [ ] إلغاء أمر إيراد منشور (FULL_CANCEL)
- [ ] تخفيض مبلغ أمر منشور (REDUCTION)
- [ ] ربط أمر الإلغاء بالأمر الأصلي
- [ ] طباعة Annexe 3 و 4

### 2.5 بوردرو الإرسال (Annexe 5)
- [ ] تجميع أوامر مُصدَرة في بوردرو واحد
- [ ] حساب المجموع الكلي تلقائياً
- [ ] طباعة Annexe 5 وإرساله لأمين الخزينة

### 2.6 تقارير واستعلامات
- [ ] قائمة أوامر الإيراد مع فلاتر (السنة، الحالة، المدين)
- [ ] إجمالي الإيرادات حسب محور الميزانية
- [ ] سجل العمليات (audit_log)

## 3. المتطلبات غير الوظيفية (Non-Functional Requirements)

| المعيار | المتطلب |
|---|---|
| **الأداء** | فتح القوائم < 500ms، طباعة PDF < 3s |
| **الموثوقية** | قاعدة بيانات MariaDB4j مع نسخ احتياطية GZIP |
| **الأمان** | كلمة مرور مُشفّرة (BCrypt) في قاعدة البيانات |
| **الاستخدام** | واجهة عربية RTL، خطوط عربية واضحة |
| **التوافق** | Windows 10/11 مع Java 21 |
| **الصيانة** | فصل واضح بين Controller / Service / Repository |

## 4. قواعد العمل المستخرجة من التعليمة 08/2023

| الرقم | القاعدة | مصدر التحقق |
|---|---|---|
| BR-01 | رقم الأمر يجب أن يكون فريداً ضمن السنة المالية | UNIQUE KEY في قاعدة البيانات |
| BR-02 | المبلغ يجب أن يكون > 0 | Guard في `RevenueOrderService` |
| BR-03 | لا يمكن إلغاء أمر في حالة DRAFT | Guard في `CancellationOrderService` |
| BR-04 | لا يمكن تخفيض المبلغ لأكثر من المبلغ الأصلي | Guard مع DECIMAL(18,2) |
| BR-05 | البوردرو لا يمكن أن يحتوي أوامر من سنوات مختلفة | Guard في `DispatchSlipService` |
| BR-06 | كل أمر في البوردرو يجب أن يكون في حالة ISSUED | Guard في `DispatchSlipService` |
| BR-07 | المبلغ بالحروف إلزامي على النسخة المطبوعة | TafqeetJ قبل الطباعة |

## 6. حدود النظام (Out of Scope - v1.0)

- لا يوجد تكامل مع منظومة IBIS أو أنظمة وطنية
- لا يوجد تطبيق ويب أو موبايل
- لا توجد صلاحيات متعددة المستوى (RBAC) — مستخدم واحد في v1.0
- لا يوجد تكامل مع بريد إلكتروني أو توقيع رقمي
